# MVP残タスク一覧（優先順つき）

最終更新: 2026-02-12（JST）

## 1. 目的

- MVP開始前に「未実装箇所」と「着手順」を固定する
- 次セッションで、そのまま上から実行できる実務用バックログにする

## 2. 対象範囲

- 対象: MVPスコープ内の未実装/未完了項目
- 除外: 第2段階（SNS/AI: X API, Gemini, Grok比較PoC）

## 3. 優先度定義

1. P0: BOTのMVP運用成立に直結する必須タスク
2. P1: 運用効率・管理性を高めるタスク（MVP内）
3. P2: 仕様確定・運用ルール明文化タスク

## 5. 仕様未確定ポイント（P2）の対象

1. 通知時刻「21時」の運用基準（JST固定か、ユーザーTZ追従か）
2. PER未定義時の振る舞い（PSRフォールバック or 通知なし）
3. データ取得フォールバック優先順の正式固定
4. 「来週」の範囲（週起点）
5. LINE通知方式（Messaging API前提の最終確定）

注記:

- 先行実装の暫定値は `MVP仕様分解_受け入れ基準.md` 7章を使用する。
- 正式確定した項目は同ファイルの確定章へ移管し、未確定章から削除する。

# 株式監視BOT MVP仕様（現行運用版）

最終更新: 2026-03-01（JST）

このドキュメントは、運用中のMVP仕様だけを残した正本です。  
完了済みの受け入れ基準（Given/When/Then）の詳細はアーカイブを参照してください。

- アーカイブ: `docs/90_アーカイブ/2026-02-18/MVP仕様_受け入れ基準詳細_2026-02-18時点.md`

## 1. 現行MVPスコープ

1. ウォッチリスト管理（最大100銘柄）
2. PER/PSR 日次計算（終値ベース）
3. 期間中央値判定（1W/3M/1Y）
4. 通知（PER/PSR、決算、データ不明）
5. Discord配信
6. 同条件通知のクールダウン（デフォルト2時間、全体設定で変更可能）
7. フェイズA保有ブリーフ通知（`--enable-phase-a` 指定時）

## 2. 固定ルール

1. 対象は日本株のみ（ETF/REIT除外）
2. ティッカーは `1234:TSE` 形式
3. 期間定義は営業日ベースで `1W=5 / 3M=63 / 1Y=252`
4. 強通知は `under_1y && under_3m && under_1w`
5. 通知時刻は `notify_timing`（`IMMEDIATE` / `AT_21` / `OFF`）で制御
6. 時刻基準はJST固定

## 3. 通知カテゴリ

- `PER割安`
- `超PER割安`
- `PSR割安`
- `超PSR割安`
- `PER状況`
- `PSR状況`
- `今週決算`
- `明日決算`
- `データ不明`
- `フェイズA保有`（オプトイン）

## 4. 欠損時ポリシー

1. サイレント失敗は禁止
2. EPS/売上/決算日時の欠損は `【データ不明】` で通知
3. PER未定義日は、PER監視銘柄でもPSRへ自動切替しない

## 5. データ取得優先順

1. 株探
2. Yahoo!ファイナンス

補足: `JQUANTS_API_KEY` が設定されている場合は、市場データ取得で `J-Quants v2` を先行利用します（運用・実装詳細は運用手順を参照）。四季報onlineはサイト側のJavaScript/Cookie要件により日次取得経路から除外しています。

## 6. MVP外（第2段階）

1. IR/SNS監視の高度化
2. AI注目通知の拡張
3. フェイズB（新規銘柄リサーチ）

標準方針:

- SNS取得は `Grok API`（プロンプトベース）
- AI要約は `Vertex AI Gemini`
- SNS取得設定（実行時刻・取得間隔・プロンプト）は管理画面から変更可能とする

## 7. 変更管理ルール

1. 仕様変更はIssueに受け入れ基準を追記してから実装する
2. 振る舞い変更時は関連ドキュメントを同一PRで更新する
3. 完了済みの詳細チェックリストはアーカイブへ移し、正本は現行仕様のみを維持する

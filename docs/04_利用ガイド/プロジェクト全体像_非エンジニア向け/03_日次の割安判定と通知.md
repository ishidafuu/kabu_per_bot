# 3. 日次の割安判定と通知

## 日次処理の流れ

1. 銘柄データを取得
2. `PER/PSR` を計算
3. 期間中央値（1W/3M/1Y）と比較
4. 通知条件に合えばDiscordへ送信

## 判定の考え方（非技術向け）

- 「現在値が過去中央値より低い」状態を割安候補として扱う
- 複数期間で一致したときほど強いシグナルとして扱う
- 通常通知と強通知を分け、優先度を見分けやすくする

## 通知が送られる主な条件

- 割安条件が成立
- `always_notify_enabled=true` で割安未成立（状況通知）
- 必須データ欠損や取得失敗（`【データ不明】`）
- 通常→強への遷移

## 重複抑制（クールダウン）

同じ内容の連投を防ぐため、クールダウンが適用されます（デフォルト2時間、管理画面で変更可能）。  
ただし、通知強度が上がった場合はクールダウン中でも通知されることがあります。

## 実行時間（JST）

- 平日18:00: `IMMEDIATE` 銘柄の判定
- 平日21:05: `AT_21` 銘柄の判定

## 通知0件が正常なケース

ジョブ成功でも、当日条件に合う銘柄がなければ通知0件は正常です。  
障害判定は、ジョブ失敗や `【データ不明】` の増加傾向とあわせて行います。

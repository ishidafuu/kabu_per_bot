# 9. 現時点の実装状況（運用確認ガイド）

最終更新: 2026-02-18（JST）

このページは、固定のテスト件数を示すスナップショットではなく、
運用時に「いま健全に動いているか」を確認するための観点をまとめたものです。

## 1. 主要機能の実装状態

- ウォッチリスト管理（登録・更新・削除・履歴）
- 日次判定（PER/PSR、中央値、under判定、連続日数）
- 通知（割安/強通知、決算、データ不明、クールダウン）
- Web管理画面（ダッシュボード、履歴、通知ログ、運用操作）

## 2. 定期確認するポイント

1. バックエンドテストが通るか
2. E2E（UI/API結合）が通るか
3. 日次ジョブ実行ログで失敗が増えていないか
4. `【データ不明】` 通知が急増していないか
5. 期待したカテゴリの通知が `notification_log` に保存されているか

## 3. 判断の目安

- テストが通っていても、外部データ欠損やAPI制限で運用品質は変動する
- 「ジョブ成功」だけでなく、通知件数傾向と欠損傾向を合わせて判断する
- 異常が続く場合は、`docs/03_運用/` の手順で切り分けを行う

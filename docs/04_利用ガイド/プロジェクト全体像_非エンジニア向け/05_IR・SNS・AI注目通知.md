# 5. IR・SNS・AI注目通知（現行仕様）

最終更新: 2026-02-19（JST）

このページは、トレーダー確認用に「現在の実装どおり」の挙動をまとめたものです。

## 1. 通知カテゴリの役割

- `IR更新`: 新着リンクの検知通知（速報）
- `SNS注目`: X（旧Twitter）投稿の検知通知（定時判定）
- `AI注目`: 取得した本文を要約した通知（内容把握）

要点:

- `IR更新` は「新着を即時把握するため」の通知です。
- `SNS注目` は「定時に新着投稿を拾って把握するため」の通知です（常時リアルタイム監視ではありません）。
- `AI注目` は「本文の要点を把握するため」の通知です。
- 同じイベントで `IR更新` と `AI注目` の両方が送られるのが現行仕様です。

## 2. AI注目の有効条件

ウォッチリスト銘柄は、`AI注目` の常時対象です。  
AI解析に失敗した場合のみ、`AI注目` をスキップして `IR更新` / `SNS注目` を継続送信します。

## 3. IR本文の取得仕様（今回反映）

### 3.1 入力URLがHTMLページの場合

- ページ内のリンク（`<a href=...>`）を抽出
- IR関連度で優先順位を付けて上位のみ採用（既定 `max_events_per_url=5`）
- 採用リンク先の本文を取得して要約対象にする

優先度判定の主な要素:

- URLがPDFか
- タイトルに `決算` `決算短信` `説明資料` `適時開示` などが含まれるか
- URLに `/ir/` `/results` `/library` `.pdf` などが含まれるか
- `contact` `recruit` `privacy` など非IR寄り要素は減点

### 3.2 入力URLがPDF直リンクの場合

- そのURL自体を1件の `IR更新` イベントとして扱う
- PDF本文を抽出し、`AI注目` の要約対象にする

### 3.3 本文抽出の詳細

- HTML: タグ除去（script/style除去後にテキスト化）
- PDF: `pypdf` で抽出（既定 先頭8ページ、最大3000文字）
- 抽出失敗時: タイトル文字列へフォールバック

## 4. SNS注目の取得仕様（現行）

- 取得方式: X API直叩きではなく、`Grok API` の `x_search` 経由で投稿候補を抽出
- 実行条件:
  - `grok_sns.enabled=true` のときのみ実行
  - JSTの `grok_sns.scheduled_time` と一致した分のみ実処理
- 再取得抑制:
  - 同一銘柄で `SNS注目` 通知が `per_ticker_cooldown_hours` 内にある場合、再取得をスキップ
- 補足:
  - 既定値では `grok_sns.enabled=false` のため、設定しないと `SNS注目` は動作しない
  - 投稿0件は異常扱いせず、通知なしで終了
## 5. AI要約の生成仕様

- モデル: `Vertex AI Gemini`
- 標準ジョブ（`scripts/run_intelligence_job.py`）は Gemini アナライザーを使用
- 既定値: `VERTEX_AI_LOCATION=global`
- 既定値: `VERTEX_AI_MODEL=gemini-2.0-flash-001`
- 応答形式: JSON固定（`responseMimeType=application/json`）
- 失敗時挙動: Gemini認証/API/解析失敗時は `AI注目` のみスキップし、`IR更新` は継続送信

`AI注目` の出力項目:

- summary（要約）
- evidence_urls（根拠URL）
- 分類ラベル（IR/SNS）
- tone（ポジ/ニュートラル/ネガ）
- confidence（High/Med/Low）

## 6. 重複抑制・再送条件

- 同一イベントの再送防止: IR/SNSイベントは `ticker + kind + url` をキーに既読管理
- 同じURLは再通知されにくい設計
- 通知クールダウン: 同一カテゴリ・同一条件は抑制（デフォルト2時間、管理画面で変更可能）

補足:

- 未来時刻でテスト実行すると、クールダウンを超えた再送確認が可能です。

## 7. 通知文面の例（実測）

`IR更新`（速報）:

```text
【IR更新】9504:TSE 北海道電力
📝 決算短信（2026年2月6日・公認会計士等による期中レビューの完了） [PDF:411KB] PDFファイルを開きます。
🔗 URL: https://www.hepco.co.jp/corporate/ir/ir_lib/pdf/kessan2025-3q_review.pdf
🏷️ 種別: IRサイト
```

`AI注目`（本文要約）:

```text
【AI注目】9504:TSE 北海道電力 北海道電力の2026年3月期第3四半期決算短信。売上高は減少も営業利益は増加。通期業績予想は据え置き。
🔗 根拠：https://www.hepco.co.jp/corporate/ir/ir_lib/pdf/kessan2025-3q_review.pdf
🏷️ 分類：IR=決算資料 SNS=該当なし トーン=ニュートラル
🎯 確信度：High
```

`SNS注目`（定時判定 + 要約）:

```text
【SNS注目】9504:TSE 北海道電力
📝 @Official_HEPCO
💬 要約: 泊発電所の電源確保対策について動画で案内。関連ページへの導線を提示。
🔗 URL: https://x.com/Official_HEPCO/status/2019327450296299809
🏷️ 種別: 公式
```

## 8. 既知の制約

- 画像だけのPDFは本文抽出精度が下がる場合がある
- IRサイトのHTML構造次第で抽出ノイズが混ざる場合がある
- `SNS注目` は定時取得のため、投稿直後に必ず通知されるわけではない
- `AI注目` は投資判断の補助情報であり、最終判断は人が行う

## 9. トレーダー確認ポイント（確認依頼）

次の観点で運用判断をお願いします。

- `IR更新` と `AI注目` の2通同時通知を維持するか
- `IR更新` のみ簡素化して `AI注目` を主通知に寄せるか
- `AI注目` に追加したい定型項目（例: 売上/利益/ガイダンス差分）

# 7. API（システム連携）でできること

## APIの位置づけ

APIは、管理画面以外の仕組みと連携するための入口です。  
将来的に社内ツールや自動レポートと連携しやすくする目的があります。

## 提供される主な機能

- ヘルスチェック
- ウォッチリストの追加・更新・削除
- ウォッチリスト履歴の参照
- 通知ログの参照
- ダッシュボード集計値の取得

## 認証の考え方

- ヘルスチェック以外は認証付き
- 基本はFirebase IDトークンで利用者を識別

## 非エンジニア視点での利用場面

- 既存社内ツールに通知ログを取り込む
- ダッシュボード数値を別レポートへ転記自動化する
- 運用監査向けに履歴を定期取得する

## 注意点

- APIを直接使う場合でも、利用権限の管理は必須
- 仕様変更時は利用側処理の追従が必要

# 6. 管理画面（Web）でできること

## 主な画面

- ログイン
- ダッシュボード
- ウォッチリスト一覧
- ウォッチリスト履歴
- 通知ログ
- 使い方ガイド

## 画面ごとの役割

- ダッシュボード: 全体の監視状況を短時間で把握
- ウォッチリスト: 監視対象と設定を更新
- 履歴: 変更理由・変更者・変更日時を確認
- 通知ログ: 通知漏れ・過剰通知の分析

## 非エンジニア担当が見るべき順番

1. ダッシュボードで当日の全体状況を確認
2. 通知ログで異常増減を確認
3. 必要時にウォッチリスト設定を修正
4. 履歴で修正の影響範囲を追跡

## 期待される効果

- 設定変更の属人化を減らす
- 通知トラブル時の原因調査を早くする
- 運用引き継ぎ時の理解コストを下げる

# kabu_per_bot 管理画面 使い方手順書

最終更新: 2026-02-18（JST）

この手順書は管理画面の `使い方` ページ（`/guide`）と同等内容です。

## 0. はじめに（このガイドで分かること）

1. 「どの画面で、どの数字を見ればよいか」をトレーダー向けに整理したガイドです。
2. 特に、ダッシュボードとウォッチリストの各項目の意味を重点的に説明します。
3. まずは「1. 3分で始める」と「2. ダッシュボードの見方」を読めば、日次運用を開始できます。

## 1. 3分で始める（最短手順）

1. 管理画面へアクセス: `https://kabu-per-bot-487501.web.app`
2. ログイン画面でメールアドレス/パスワードを入力し、「Firebaseでログイン」を押します。
3. ダッシュボード右上の「ウォッチリストへ」を押します。
4. 「新規追加」を押して、例として `7203:TSE`（トヨタ自動車）を登録します。
5. 監視方式は迷ったら `PER`、通知時間は迷ったら `IMMEDIATE` を選びます。
6. 保存後、一覧に1件表示されたら初期設定は完了です。

## 2. ダッシュボードの見方（数字の意味）

1. `監視銘柄数`:
   登録中の銘柄件数です。100件が上限です。想定より少ない場合は、誤って削除していないか履歴で確認します。
2. `当日通知件数`:
   当日JSTで送信された通知件数です。0件でも、条件未達なら正常です。
3. `データ不明件数`:
   当日JSTで `【データ不明】` として通知された件数です。0以外なら、該当銘柄のデータ取得失敗が起きています。
4. `失敗ジョブ有無`:
   日次系ジョブに失敗があったかを `あり / なし` で表示します。`あり` の場合は優先確認対象です。
5. `再読み込み`:
   画面表示中に最新値を取り直すボタンです。状況確認の最初に押すと安全です。

## 3. ウォッチリスト「入力項目」の意味（追加・編集）

1. `ticker`:
   銘柄コード。必ず `1234:TSE` 形式で入力します（例: `7203:TSE`）。
2. `会社名`:
   一覧で見分けるための表示名です。略称でも構いませんが、チーム内で統一してください。
3. `監視方式`:
   `PER` または `PSR`。その銘柄で主に見たい評価軸を選びます。
4. `通知時間`:
   `IMMEDIATE`（日次評価）/ `AT_21`（21時評価）/ `OFF`（通知しない）です。
5. `常時通知（割安でない場合も通知）`:
   ONにすると、割安条件が成立しない日でも状況通知が届きます。監視を切らしたくない銘柄向けです。
6. `有効`:
   OFFにすると監視対象から外れます（登録は残るため、再開しやすい）。
7. `AI通知`:
   AI要約通知を受け取る設定です。運用側で全体設定がONのときのみ有効になります。

## 4. ウォッチリスト「一覧列」の意味（日々ここを見る）

1. `監視指標`:
   その銘柄で判定に使う軸（PER/PSR）です。
2. `通知タイミング`:
   いつ通知判定されるかを示します（IMMEDIATE / AT_21 / OFF）。
3. `常時通知`:
   `true` は状況通知あり、`false` は条件一致時のみ通知です。
4. `有効状態`:
   `true` は監視中、`false` は停止中です。
5. `現在値`:
   最新の PER/PSR 値です。`-` は当日値を取得できていない状態です。
6. `中央値（1W/3M/1Y）`:
   直近1週間・3か月・1年の基準値です。現在値と比較して割安判定に使われます。
7. `シグナル`:
   割安条件の成立状況です。`(n日連続)` は連続成立日数を示します。
8. `次回決算`:
   決算予定日と時刻です。`未定` は時刻未公表を意味します。

## 5. 通知ログの見方（通知の追跡）

1. `送信日時`:
   通知が送られた時刻（JST）です。
2. `通知カテゴリ`:
   どの種類の通知か（割安系、決算系、データ不明など）を示します。
3. `強通知`:
   `true` の場合は、通常より優先度の高い通知です。
4. `条件キー`:
   どの条件で通知されたかを識別するキーです。重複通知確認に使います。
5. 銘柄単位で確認したい場合は、検索欄に `7203:TSE` のように入力して絞り込みます。

## 6. 履歴の見方（誰が何を変えたか）

1. `操作日時`:
   追加・削除・編集が行われた時刻です。
2. `操作種別`:
   何を実施したか（追加/削除など）を示します。
3. `理由メモ`:
   変更の背景メモです。後から判断理由を追うために活用します。
4. 銘柄別に見たい場合は ticker で絞り込みます。

## 7. よくあるトラブル（先にここを見る）

1. 通知が来ない:
   `通知時間=OFF` になっていないか、`有効=false` になっていないかを確認します。
2. 通知件数が少ない:
   条件未成立の日は0件でも正常です。必要なら `常時通知=true` を検討します。
3. `現在値` が `-` のまま:
   データ取得遅延や欠損の可能性があります。通知ログで `【データ不明】` を確認します。
4. 画面に想定銘柄がない:
   検索条件の絞り込みが残っていないか確認し、履歴で削除操作の有無を見ます。

## 8. 日々の運用チェック（おすすめ）

1. ダッシュボードで `失敗ジョブ有無` を確認（まずはここ）。
2. ウォッチリストで `有効状態=false` が混ざっていないか確認。
3. 通知ログで当日の主要銘柄通知を確認。
4. 銘柄を変更した日は履歴に理由メモを残す。

## 9. 用語ミニ辞典

1. `PER`:
   株価が1株利益の何倍かを示す指標です。
2. `PSR`:
   株価が1株売上の何倍かを示す指標です。
3. `割安判定`:
   現在値が過去中央値より低い状態を組み合わせで評価したものです。
4. `クールダウン`:
   同条件通知の短時間連投を防ぐ仕組みです（2時間）。

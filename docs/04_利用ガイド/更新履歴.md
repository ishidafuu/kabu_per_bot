## 4. 運用ルール

1. 利用者向けの表示や挙動が変わる変更を行ったら、この履歴へ同日追記する。
2. 1行は「何が変わったか」を短く書き、専門実装の詳細は別ドキュメントやPRに委譲する。
3. 同日の複数変更は、`機能追加` / `バグ修正` をまたいで分けて記録する。

## 1. このプロジェクトの全体像

このプロジェクトは、**指定した日本株を毎日チェックし、条件に当てはまったらDiscordに通知する仕組み**です。  
主な利用シーンは「監視銘柄を定期チェックして、見逃しを減らす」ことです。

大きく分けて、次の2つがあります。

- BOT本体: データを集めて判定し、通知を送る
- 管理画面: 監視銘柄や履歴を人が確認・編集する

---

## 2. ウォッチリスト管理（銘柄の登録・更新）

ウォッチリストは「どの銘柄を監視するか」を管理する一覧です。

できること:

- 銘柄を新規追加する
- 銘柄情報を編集する
- 銘柄を削除する
- 最大100銘柄まで管理する
- 追加・削除した履歴を残す（理由メモ付き）

設定できる主な項目:

- 監視指標: PER または PSR
- 通知先: Discord か 通知OFF
- 通知タイミング: すぐ通知 / 21時通知 / 通知OFF
- 銘柄の有効・無効
- AI注目通知の有効・無効
- IRページURL、X（旧Twitter）アカウント

---

## 3. 日次の割安判定と通知

毎日のジョブで、登録銘柄ごとに次を実行します。

1. 株価や予想値などの必要データを取得
2. PER/PSRを計算
3. 1週間・3か月・1年の中央値と比較
4. 条件に当てはまれば通知

通知のポイント:

- 条件に応じて「割安」通知を送る
- 強い条件に当てはまる場合は強通知（例: 超PER割安）
- 同じ内容の連続通知を避けるため、**2時間クールダウン**を適用
- データ不足や取得失敗時は、必ず `【データ不明】` 通知を送る
- 通知タイミング設定により、`すぐ通知(IMMEDIATE)` と `21時通知(AT_21)` のどちらで送るかを切り替える

これにより、通知漏れだけでなく「判定不能だった事実」も把握できます。

## 4. 決算通知ジョブ

決算関連は、用途別に2種類の通知があります。

- 今週決算: 来週予定の決算をまとめて通知
- 明日決算: 翌日の決算を通知

運用イメージ:

- 週次・日次で実行し、重要イベントを前もって把握する
- ウォッチリストにある銘柄だけを対象にする

---

## 5. IR・SNS・AI注目通知

割安通知とは別に、情報更新の検知通知があります。

できること:

- IRページの更新検知（IR更新）
- Xアカウント投稿の検知（SNS注目）
- 必要に応じてAI注目通知を追加

補足:

- 同じ投稿・同じURLの重複通知は抑制
- AI注目は「全体設定ON」かつ「銘柄側ON」のときのみ有効

---

## 6. 管理画面（Web）でできること

管理画面では、次のページを利用できます。

- ログイン
- ダッシュボード
- ウォッチリスト
- ウォッチリスト履歴
- 通知ログ
- 使い方ガイド

実務で便利な点:

- ウォッチリスト一覧で最新値・判定・次回決算をまとめて確認
- 履歴で「いつ誰が何を変更したか」を追跡
- 通知ログで「何がいつ通知されたか」を確認

---

## 7. API（システム連携）でできること

外部システムや将来の拡張向けに、APIも用意されています。

主なAPI:

- ヘルスチェック
- ウォッチリスト CRUD
- ウォッチリスト履歴一覧・理由更新
- 通知ログ一覧
- ダッシュボード集計

認証:

- 基本的にログインユーザー（Firebase IDトークン）向け
- ヘルスチェックのみ公開

---

## 8. 運用を支える補助機能

本番運用を支えるため、次の補助機能があります。

- Firestore初期セットアップ（マイグレーション）
- Discord疎通確認（テスト通知）
- デプロイ前チェック（設定漏れ・ビルド確認）
- WebのE2E用テストAPI

これにより、設定ミスや反映ミスを事前に見つけやすくなっています。

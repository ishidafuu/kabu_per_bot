## 2. 通知が出る主条件

日次:

- 割安成立（`1Y+3M` / `3M+1W` / `1Y+1W` / `1Y+3M+1W`）
- `always_notify_enabled=true` で割安未成立（`PER状況` / `PSR状況`）
- 欠損時（`データ不明`）

決算:

- 抽出条件に合う決算日があり、銘柄設定が通知対象

IR/SNS:

- 新規イベント（既読でない）を検知
- AIは「全体ON + 銘柄ON」で `AI注目` を追加

---

## 3. 通知が来ない主理由（優先確認順）

1. ジョブ未起動  
Scheduler停止、Job未作成、Secret不足

2. 起動したが実処理ゲートで停止  
IMMEDIATE帯外/間隔不一致、Grok定時不一致

3. 銘柄が通知対象外  
`is_active=false` / `notify_channel=OFF` / `notify_timing` 不一致

4. 条件未成立  
割安未成立（かつ常時通知OFF）、該当決算なし、新規IR/SNSなし

5. 通知抑制でスキップ  
クールダウン、既読、IR初回既読化、公開日範囲外、Grok銘柄別取得抑制

---

## 4. 抑制ロジック（最小要点）

- 共通クールダウン:
  - 同一 `ticker + category + condition_key` がクールダウン時間内なら送信しない
  - ただし通常→強遷移は即時送信
- IR固有:
  - 銘柄初回IRは通知せず既読化のみ
  - 古い公開日（`intel_notification_max_age_days` 超過）は通知しない
- Grok固有:
  - `SNS注目` 直近通知が `per_ticker_cooldown_hours` 内なら再取得しない

---

## 5. カウンタの読み方

- `processed`: 銘柄処理に入った件数
- `sent`: 実送信件数
- `skipped`: 通知候補が抑制された件数
- `errors`: エラー件数

`sent=0` でも、条件未成立なら正常です。
